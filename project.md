In this project, you are required to design and implement a computer game using OpenGL.
You may choose any game idea that interests you, but you must enter the name of your
chosen game in the provided Excel sheet. No two teams can work on the same game, so
priority will be given to teams who submit their game ideas first. Projects will be completed
in teams of two students. Teams submitting duplicate ideas will face a deduction in marks.
■ Bonus: Teams that develop a 3D game using OpenGL will receive bonus points.

Updated Project Phases and Requirements

Phase Deadline
Single Project Phase Week 10

Submission Requirements:
• Game code.
• Screenshot of your current game interface or design.
• A comment in your submission file that includes:
• • Names of team members
• • Their registration numbers
• • The name of the Teaching Assistant (TA) you're registered with

Minimum Functional Requirements:
1. Timer – A non-interactive or interactive text-based timer must be displayed on the
screen.
2. Score Display – A non-interactive or interactive score in text format.
3. Patterned Background – Design a background using a custom or procedural pattern to
enhance visual aesthetics.
4. Player Object – Create a player character composed of at least three OpenGL shapes
(e.g., circle for face, rectangle for body, etc.). The player must be movable using keyboard
input (e.g., arrow keys or WASD keys).
5. Main Menu – A fully functional main menu with OpenGL-rendered buttons for Start
Game and Exit.
6. Interactive Score and Timer – Implement logic to increase/decrease score based on
in-game events. Timer should now be interactive, counting down or controlling game over conditions.
